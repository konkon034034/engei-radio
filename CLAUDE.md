# engei-radio プロジェクト仕様書

## AI作業ルール
- **Thinking（思考プロセス）は必ず日本語で書く。英語は絶対禁止**
- 未使用コードの残骸は即座に削除する

### 会話引き継ぎルール
- **task.mdに作業状態を常に記録する**
- **walkthrough.mdに完了作業の成果を記録する**

## 概要
もう枯らさない家庭園芸 チャンネル。DynamicNewsVideo形式の動画を自動生成。

## 動画形式
DynamicNewsVideo（G1統一コンポーネント）

### 3幕構成
| 幕 | 内容 |
|:---|:---|
| **OP** | クイズ/衝撃データ/悩み相談の3パターンから自動選択 |
| **本編** | DataOverlay(チャート)を使った解説。カツミとヒロシの掛け合い |
| **控え室** | エピローグ余韻。温かいトーク |

## 絶対ルール
1. **動画生成はRemotion専用**（MoviePy禁止）
2. **動画は3分以上**
3. **TTS優先順位**: Edge TTS → Amazon Polly
4. **カツミ・ヒロシの人間性**: 体験談・共感・庶民あるある

### 台本テスト（2026-02-20 追加）

> [!IMPORTANT]
> 「台本テスト」と言われたら以下を実行する。

- **目的**: 動画の構成と台本のテキスト表示品質を確認する
- **最重要**: 人間は文字によってのみ教育される。画像やグラフはサポート。しゃべっている言葉の内容とそれを画面にテキストでしっかり表示しているかが大事
- **形式**: 紙芝居形式でOK。音声なし。Remotion静止画でOK
- **ダミー許可**: 画像/グラフ/音声/チャートは全てダミーでOK
- **本番必須**: 台本(セリフ・字幕テキスト)だけは本番と同じクオリティ
- **出力**: 各シーン(OP/本編/控え室)の台本テキストをアスキーアート風レイアウトで表示。Remotion静止画(npx remotion still)も併用
- **頻度**: 今後は基本的に台本テストを多用する
